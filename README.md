# TG WS Proxy

**TG WS Proxy** — локальный MTProto-прокси для Telegram Desktop, использующий WebSocket как транспортный уровень передачи данных. Такой подход позволяет направлять трафик напрямую к серверам Telegram без сторонних MTProto-прокси, сохраняя стандартный механизм шифрования протокола.

# Установка

Перейдите в раздел **Releases** репозитория и скачайте сборку, подходящую для вашей операционной системы.

Доступны следующие версии:

- `TgWsProxy_windows.exe` — Windows 10 и новее (x64);
- `TgWsProxy_windows_arm64.exe` — Windows 10 и новее (ARM64);
- `TgWsProxy_windows_7_64bit.exe` — Windows 7 (64-bit);
- `TgWsProxy_windows_7_32bit.exe` — Windows 7 (32-bit).

Во время первого запуска приложение автоматически откроет инструкцию по подключению Telegram Desktop. После завершения настройки программа продолжит работу в фоновом режиме, а доступ к её функциям будет осуществляться через значок в области уведомлений.

## Меню программы

Все основные действия доступны через контекстное меню TG WS Proxy.

Доступные пункты:

- **Открыть в Telegram** — автоматически сформировать ссылку `tg://proxy` и открыть её в клиенте Telegram.
- **Скопировать ссылку** — сохранить ссылку подключения в буфер обмена.
- **Перезапустить прокси** — повторно запустить локальный MTProto-прокси без завершения работы приложения.
- **Настройки...** — открыть окно конфигурации, изменить параметры работы и настроить автоматическую проверку обновлений.
- **Открыть журнал** — перейти к журналу событий приложения.
- **Выход** — остановить прокси и полностью закрыть программу.

# Настройка Telegram Desktop

## Автоматическое подключение

Рекомендуемый способ — воспользоваться автоматической настройкой.

Для этого:

1. Щёлкните правой кнопкой мыши по значку TG WS Proxy.
2. Выберите **«Открыть в Telegram»**.
3. Подтвердите подключение прокси.

Если Telegram не обработал ссылку автоматически:

1. Откройте меню приложения.
2. Нажмите **«Скопировать ссылку»**.
3. Отправьте её в чат **«Избранное»**.
4. Откройте полученную ссылку.
5. Подтвердите подключение.

## Ручная настройка

Если автоматический способ недоступен, параметры можно задать самостоятельно.

Откройте:

```text
Telegram
→ Настройки
→ Продвинутые настройки
→ Тип подключения
→ Прокси
```

Создайте новое MTProto-подключение и заполните поля следующими значениями:

| Параметр | Значение |
|----------|----------|
| Тип | MTProto |
| Сервер | `127.0.0.1` (или адрес, указанный в настройках TG WS Proxy) |
| Порт | `1443` (либо выбранный пользователем порт) |
| Secret | секретный ключ, отображаемый в настройках приложения или журнале логов |
# Принцип работы

```text
Telegram Desktop
        │
        ▼
Локальный MTProto Proxy
        │
        ▼
WebSocket (TLS)
        │
        ▼
Telegram Data Center
```

После запуска TG WS Proxy начинает принимать подключения от Telegram Desktop, определяет необходимый дата-центр Telegram и перенаправляет трафик через защищённое WebSocket-соединение.

Основные этапы обработки подключения:

1. Запускается локальный MTProto-прокси с параметрами, указанными в конфигурации приложения.
2. Принимаются входящие подключения от Telegram Desktop.
3. Анализируется **MTProto Obfuscation Init Packet**, из которого определяется идентификатор требуемого дата-центра (DC ID).
4. Создаётся защищённое WebSocket-соединение с соответствующим сервером Telegram.
5. Если подключение через WebSocket установить не удаётся (например, сервер отвечает кодом **HTTP 302 Redirect**), TG WS Proxy автоматически переключается на резервный маршрут, используя Cloudflare Proxy или прямое TCP-соединение.

---

> [!IMPORTANT]
>
> ### После подключения перестали загружаться медиафайлы?
>
> Если после включения TG WS Proxy Telegram перестал отображать изображения, видео или другие вложения, в первую очередь проверьте настройки IP-адресов дата-центров.
>
> Откройте раздел:
>
> **DC → IP**
>
> и оставьте только следующую запись:
>
> ```text
> 4:149.154.167.220
> ```
>
> Если проблема остаётся, полностью очистите содержимое этого поля и повторите подключение.
>
> Иногда подобная ситуация наблюдается у пользователей без активной подписки **Telegram Premium**.
>
> Если проблема сохраняется, рекомендуется использовать собственный Cloudflare Proxy, выполнив настройку по инструкции:
>
> **[CfProxy.md](./CfProxy.md)**

# Автоматическая сборка

Репозиторий уже содержит всё необходимое для автоматического формирования исполняемых файлов.

В качестве инструмента упаковки используется **PyInstaller**, а создание новых сборок полностью автоматизировано при помощи **GitHub Actions**.

Используемые спецификации находятся в следующих файлах:

- `packaging/windows.spec`
- `packaging/macos.spec`
- `packaging/linux.spec`

Workflow сборки расположен по пути:

- `.github/workflows/build.yml`

Такая схема позволяет автоматически получать актуальные сборки после внесения изменений в репозиторий.
